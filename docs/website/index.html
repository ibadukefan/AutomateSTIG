Why AutomateSTIG?

⚡

Lightning Fast

Populate checklists from SCC/ACAS/OpenSCAP scans in seconds, not hours. Evaluate hundreds of rules across multiple assets in a single command.

🔒

Air-Gapped by Design

No network calls, no auto-updates, no external services. Import signed .stigpack files from USB/DVD. The only mode is offline mode.

📄

Modern Answer Files

JSON/YAML answer files replace tedious XML. Version-control your templates, bulk import/export, and generate templates from existing checklists.

🛠

Built-in Remediation

Ansible playbooks, PowerShell scripts, and Bash scripts for common fixes. One-click "apply & re-scan" for supported platforms.

🚀

Cross-Platform

Native on Windows, macOS, and Linux. Single binary — CLI (6.5 MB) or Desktop GUI (16 MB). No PowerShell dependency. No runtime requirements.

📈

Enterprise Integrations

STIG-Manager bidirectional sync (push results, pull assets). eMASS CSV export. Webhook notifications. Scheduled evaluations with drift detection.

💻

Remote Scanning

SSH + WinRM remote data collection. Parallel batch evaluation. Asset inventory with encrypted credential vault. Kerberos, SSH key, token, and certificate auth.

🕒

Scheduled Compliance

Cron-like evaluation schedules — daily, weekly, monthly, or custom intervals. Auto-push to STIG-Manager. Alert on CAT I findings or compliance drift.

AutomateSTIG vs. Evaluate-STIG

AutomateSTIG is being built to be a validated, better replacement for Evaluate-STIG. Replacement-ready claims are tied to coverage manifests, real scanner/checklist fixtures, and CI evidence rather than marketing breadth.

Architecture

Core Engine

Fully data-driven STIG evaluation. All rules, checks, and mappings live in a versioned STIG Library. The engine is thin, deterministic, and rarely needs updates.

crates/core

Parsers

Read/write CKL, CKLB, XCCDF/SCAP, and device config dumps with full fidelity. Roundtrip-safe — what goes in comes back out identical.

crates/parsers

STIG Packs

Signed .stigpack archives containing benchmarks, answer templates, and remediation scripts. SHA-256 integrity + Ed25519 signatures.

crates/stigpack

Storage

SQLite database for evaluation history, audit logs, and cached checklists. Single-file, portable, no server required.

crates/storage

Integrations

STIG-Manager API with Result Engine metadata. eMASS CSV export. Designed for seamless enterprise workflows.

crates/integrations

Remediation

PowerShell, Bash, and Ansible scripts with risk levels, admin requirements, and optional rollback support.

crates/remediation
